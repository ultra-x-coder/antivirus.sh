Linuxのためのマルウェアスキャン、ネットワーク監査、システム全体のハードニングをBashスクリプト1本で。

依存関係ゼロ。すべてのUbuntuバージョンと、事実上あらゆるLinuxディストリビューションで動作します。デフォルトは監査のみ、必要なときだけ対話的に修正、そしていつでも完全ロールバック。

antivirus.sh を選ぶ理由

世の中のLinuxマシンの大半はサーバーであり、侵害の入口はいつも同じ、ありふれた扉です。公開されたポート、パスワード認証のSSH、忘れ去られたcronジョブ、パッチ未適用のパッケージ。antivirus.sh はそれらの扉を数分で閉じ、すでに入り込んだ侵入者がいないかも探し出します。

📦

単一ファイル、依存関係ゼロ

Bashスクリプト1本だけ。Pythonもエージェントもデーモンもなく、コンパイルも不要です。ダウンロードして実行し、不要になれば消すだけ — 後に残るのはレポートと（お望みなら）ハードニング済みのシステムです。

どのUbuntuでも。どのLinuxでも。

すべてのUbuntuリリースで動作を保証 — 古いLTSから最新版まで。Debian、RHEL、CentOS、Alma、Rocky、Fedora、Arch、openSUSE、Alpineはベストエフォートでサポートし、すべてのチェックは利用できない環境でも安全に縮退します。

締め出されない設計

ファイアウォール有効化の前にSSHポートを事前許可。鍵が実在しない限り鍵認証のみへの切り替えは拒否。新しいsshd設定はリロード前に検証。すべてバックアップされ、--rollback で元に戻せます。

本格派ツールとの連携も万全

ClamAV、rkhunter、chkrootkitが存在すれば検出して連携し、フラグ1つでインストールも可能。antivirus.sh を高速な第一防衛線として、そしてディープスキャナーをまとめる接着剤として活用できます。

チェック項目の完全カタログ

6つの領域にわたる70以上の個別チェック。すべての検出結果はOK / INFO / WARN / CRITに分類され、最終的な0〜100のセキュリティスコアに反映されます。

01 マルウェアとルートキット

  • 既知のマイナー・マルウェアのプロセス名（xmrig、kinsing、kdevtmpfsi…）とマイニングプールのコマンドライン
  • /tmp、/var/tmp、/dev/shm から実行されているプロセス
  • バイナリが削除済みのプロセス、ファイルレスな memfd 実行ファイル
  • 隠しプロセスの検出（readdir隠蔽型カーネルルートキットのテスト）
  • 既知のルートキット（Diamorphine、Reptile…）に一致するカーネルモジュール
  • /etc/ld.so.preload とグローバルな LD_PRELOAD によるユーザーランド・ルートキット
  • tmpディレクトリと /dev 内の隠し実行ファイル
  • 2 MBまでのスクリプト内のリバースシェル／ドロッパー／マイナーのパターン
  • パッケージのチェックサムによるコアバイナリの整合性検証（dpkg -V / rpm -V）
  • ClamAV、rkhunter、chkrootkitとの連携（オプション）

02 永続化とバックドア

  • システムとユーザーのcrontab、cron.d、hourly/daily/weekly/monthly、at ジョブ
  • tmpから実行される、または攻撃パターンに一致するカスタムsystemdユニット
  • rc.local、initスクリプト、udevルール、MOTDスクリプト
  • 全ユーザーのシェル起動ファイル（.bashrc、.profile、.zshrc…）
  • 全アカウントの authorized_keys 監査 — システムアカウントに設定されたSSH鍵を警告
  • 余分なUID 0アカウント、NOPASSWDなsudoルール、レガシーな .rhosts 信頼ファイル
  • サードパーティAPTリポジトリのレビュー

03 ネットワークの露出

  • リッスン中の全TCP/UDPポートを所有プロセスとバインドアドレス付きで一覧
  • 公開状況の分類: ループバックのみか、外部から到達可能か
  • 危険なサービスを警告: Telnet、FTP、r-services、SMB、RDP、VNC、rsync、NFS、rpcbind
  • デフォルトで認証なしのデータストア: Redis、MongoDB、Elasticsearch、memcached
  • :2375 で公開されたDocker API（即座にリモートrootを奪われる）の検出
  • マイニングプール／IRCボットネットのポートへの確立済み接続
  • プロミスキャスモードのインターフェース、ARPスプーフィングの兆候
  • DNSリゾルバのレビューと /etc/hosts 改ざんの検出

04 SSHとアカウント

  • rootログイン、パスワード認証、空パスワード、X11フォワーディング
  • MaxAuthTries、ログイン猶予時間、キープアライブ、ホストベース認証、rhosts
  • fail2banの有無の確認とsshd jailの自動セットアップ
  • btmp のログイン失敗記録によるブルートフォース攻撃の痕跡
  • 空パスワードのアカウント（キー1つでロック可能）、UIDの重複
  • ログインシェルを持つシステムアカウント、sudoグループのメンバーシップのレビュー
  • パスワードエージングポリシーとpam_pwqualityによる強度の強制

05 カーネルとファイルシステム

  • 16のsysctlハードニングキー: ASLR、syncookies、リダイレクト、ソースルーティング、rp_filter、ptraceスコープ、dmesg/kptrの制限、protected links…
  • shadow、sudoers、SSHホスト鍵、crontab、GRUB設定のパーミッション
  • 厳選されたホワイトリストとパッケージ所有権の証明に基づくSUID/SGID監査
  • 誰でも書き込み可能なファイルと、stickyビットのない誰でも書き込み可能なディレクトリ
  • 所有者不明のファイル、immutableフラグ（chattr +i）によるマルウェアのロック
  • /dev/shm と /tmp のマウントフラグ（noexec、nosuid、nodev）
  • コアダンプの制限とSUIDダンプの防止

06 プラットフォームとサービス

  • 未適用のセキュリティアップデート、unattended-upgrades、EOLリリースの検出
  • カーネルパッチ適用後の再起動要否の検出
  • AppArmor / SELinuxの強制アクセス制御の状態
  • auditdによるフォレンジック証跡、journaldの永続ロギング
  • NTP時刻同期（TLSとフォレンジックの前提条件です）
  • CPU脆弱性の緩和策（Meltdown/Spectre系）
  • Docker: ソケットのパーミッション、特権コンテナ、dockerグループのメンバー
  • Secure Bootの状態、GRUBパスワードの推奨

インストールと初回実行

古典的な意味での「インストール」は不要です — ただのスクリプトですから。コマンド3つで完全なレポートが手に入ります。

ダウンロード
curl -fsSL https://raw.githubusercontent.com/TARGET_PLEVEHOLDER/antivirus.sh/main/antivirus.sh -o antivirus.sh
1 — 安全な読み取り専用監査（何も変更しません）
sudo bash antivirus.sh --audit
2 — 対話モード: 各修正を確認しながら適用
sudo bash antivirus.sh
3 — または新規VMを一括ハードニング
sudo bash antivirus.sh --harden

ヒント: rootなしでも動作します — 実行可能なチェックをすべて行い、残りはその旨を明確に表示します。git cloneでもOK: git clone https://github.com/TARGET_PLEVEHOLDER/antivirus.sh

モードとオプション

スクリプトは1本、使い方は4通り: 監査、対話、自動、フルハードニング。さらにスクリプティングやcron向けの限定スコープも用意しています。

コマンド動作
sudo bash antivirus.sh対話モード — すべての検出結果を表示し、修正を1つずつ確認します。初回実行におすすめです。
--auditレポートのみ。変更ゼロを保証 — 本番環境でも安全です。終了コード0/1/2 = 問題なし/警告/重大。cronやCIに最適です。
--fix安全な修正をすべて自動適用します。リスクの高い修正（ファイアウォール、SSH認証）には引き続き明示的な確認が必要です。
--harden--fix の全内容に加え、新規VM向けフルキット: ファイアウォール、fail2ban、unattended-upgrades、sysctl一式、auditd、パスワードポリシー、バナー、コアダンプとshmのロックダウン。さらに、きちんとした管理者ユーザーの作成も提案します。
--create-usersudoユーザーのガイド付き作成: 強力なパスワードの自動生成、SSH鍵のインストール、必要に応じて鍵認証のみのSSHへの切り替え — セッションを閉じる前にテストするようリマインドします。
--scan /var/www指定ディレクトリのマルウェアスキャン（Webのドキュメントルートは格好の隠し場所です）。
--network / --system / --malware単一の領域のみを実行します。
--quick時間のかかる処理（ディスク全体のスキャン、ClamAV）をスキップ — 数秒で完了します。
--full最も深いモード: ファイルシステム全体をスキャンし、インストール済みの全パッケージを検証します。
--rollback前回の実行で変更されたすべてのファイルを復元し、ファイアウォールの有効化も取り消します。
--install-toolsシグネチャベースの深掘りのため、ClamAV、rkhunter、chkrootkitをインストールします。
--report file.txt · --no-color · --yes · --exclude PATHレポート出力と自動化のためのヘルパーです。

自分を締め出せないように作られています

ハードニングツールには、自分が座っている枝を切り落とす悪い癖があります。antivirus.sh はリモートアクセスを何よりも神聖なものとして扱います。

  • ファイアウォール有効化はSSHを事前許可。 SSHポートを3つの方法（sshd設定、稼働中のソケット、現在の接続）で検出し、ufw enable の実行前に許可します。
  • 鍵がなければ鍵認証のみへの切り替えを拒否。 PasswordAuthentication no は、sudo権限を持つユーザーが実際に authorized_keys エントリを持っている場合にのみ適用されます。rootログインの無効化も同様です。
  • sshd設定はリロード前に検証。 すべての変更は sshd -t を通過します。検証に失敗した設定は自動的に復元され、稼働中のSSHセッションが切断されることはありません。
  • すべてをバックアップ。 変更された各ファイルは、マニフェスト付きのタイムスタンプ付きバックアップディレクトリにコピーされます。--rollback はそのすべてを復元し、スクリプトが有効化したファイアウォールの無効化まで行います。
  • 削除せず、隔離。 不審なファイルは隔離ディレクトリへ移動され、すべてのパーミッションを剥奪されます — 証拠と可逆性はそのまま保たれます。
  • リスクの高い修正は必ず確認。 --fix モードであっても、リモートアクセスに影響しうる操作には明示的な確認（または文書化された、意図的な --yes）が必要です。

レシピ: 新規VMを3分でハードニング

クラウドプロバイダーから新しいUbuntu VMを受け取り、rootパスワードが手元にある。やることは、これで全部です。

ステップ 1 — 取得してハードニング
curl -fsSL https://raw.githubusercontent.com/TARGET_PLEVEHOLDER/antivirus.sh/main/antivirus.sh -o antivirus.sh
sudo bash antivirus.sh --harden

実行中に行われること: SSHを事前許可したうえでファイアウォールを有効化 → fail2banとunattended-upgradesをインストール → 16のカーネルハードニングキーを適用 → SSHを強化（検証済み・セッション安全）→ パスワードポリシーを設定 → /dev/shm、コアダンプ、放置されたシェルをロックダウン → SSH鍵付きの個人用sudoユーザーの作成を提案 → そして鍵が配置された後にはじめて、SSHの鍵認証のみへの切り替えとrootログインの無効化を提案します。新しいログインは別のターミナルでテストしてください — 現在のセッションは最後まで生き続けます。

FAQ

ClamAVのような「本物の」アンチウイルスですか？

担う層が違います。ClamAVは何百万ものファイルシグネチャと照合します。antivirus.sh は、実際にLinuxサーバーを侵害するパターン — マイナー、リバースシェル、cronドロッパー、SSHバックドア、ルートキットの痕跡 — を狩り出し、それらの侵入を許した設定を監査します。両者は互いを補完する関係であり、だからこそ antivirus.sh はClamAVを検出し、存在すれば活用し、--install-tools であなたの代わりにインストールまでします。

古いUbuntu 14.04でも、出たばかりの最新リリースでも動きますか？

はい — それこそが設計上の制約です。純粋なBashでコンパイル済みの部品はなく、すべてのチェックにフォールバック（ss/netstat、systemd/sysvinit、ufw/iptables）があり、機能が存在しない場合はその旨を記して安全にスキップします。同じスクリプトがUbuntu 12.04世代のシステムから最新リリースまで動作します。

本番環境で実行しても安全ですか？

--audit モードは厳密に読み取り専用で、どんな環境でも最初の実行として推奨されます。変更を伴うモードは、各ステップを確認する（対話モード）か安全な修正に限定し（--fix）、触れたすべてのファイルのバックアップを保持し、--rollback を提供します。

本当にSSHから締め出されないのですか？

危険な移行 — ファイアウォールの有効化、パスワード認証の無効化、rootログインの無効化 — はすべてガードされています。SSHポートは事前に許可され、鍵認証のみへの切り替え前に鍵の存在が検証され、設定は sshd -t で検証されて失敗時には自動で元に戻り、現在のセッションが切断されることはありません。それでも新しいログインを別のターミナルでテストすることをおすすめします。スクリプトは、まさにそのタイミングでリマインドしてくれます。

CRITICALな検出がありました。どうすればいいですか？

まず検出内容を読んでください — 各項目は自らを説明し、証拠を出力します。マルウェアの検出であれば、提示された修正で隔離し、次にどうやって入り込んだのか（公開サービス? 盗まれた鍵? 古いCMS?）を考えます。重要: カーネルルートキットの兆候（隠しプロセス、ルートキットモジュール）がある場合、稼働中のシステム上でのクリーンアップは一切信頼できません — クリーンなイメージから再構築し、データのみを復元してください。

外部に通信したり、何かをアップロードしますか？

いいえ。テレメトリも、アップロードも、あなたが明示的に要求したもの（パッケージのインストール、ClamAVのシグネチャ更新）以外の外部通信もありません。レポートはあなたのディスクに留まります。挙動はすべて監査可能です — 読みやすいBashファイル1本ですから。

Lynisとの違いは何ですか？

Lynisは優れた純粋な監査ツールで、何が問題かを教えてくれます。antivirus.sh は明確な方針を持つ実践型です。アクティブなマルウェアも狩り、見つけた問題をロールバック付きで安全に修正します。両方を実行しても構いません。両者の見解が食い違うことは、ほとんどありません。

定期的に実行するには？

--audit --no-color --report を付けたcron行を追加してください（「使い方」参照）。終了コード0/1/2のおかげで、CI、cronメール、任意の監視システムからのアラートが簡単に実現できます。
