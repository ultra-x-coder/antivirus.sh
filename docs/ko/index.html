왜 antivirus.sh인가

대부분의 리눅스 머신은 서버이고, 늘 똑같이 뻔한 문으로 뚫립니다: 노출된 포트, 패스워드 SSH, 잊힌 cron 작업, 패치 안 된 패키지. antivirus.sh는 그 문들을 몇 분 안에 닫아 버립니다 — 그리고 이미 들어와 있는 침입자가 없는지도 찾아냅니다.

📦

단일 파일, 의존성 제로

Bash 스크립트 하나가 전부입니다. Python도, 에이전트도, 데몬도, 컴파일할 것도 없습니다. 내려받아 실행하고, 원하면 지우면 됩니다 — 남는 것은 보고서와 (원한다면) 하드닝된 시스템뿐입니다.

🐧

어떤 Ubuntu든. 어떤 리눅스든.

오래된 LTS부터 최신 릴리스까지 모든 Ubuntu 버전에서 동작을 보장합니다. Debian, RHEL, CentOS, Alma, Rocky, Fedora, Arch, openSUSE, Alpine은 best-effort로 지원하며, 모든 검사가 환경에 맞춰 유연하게 대체 동작합니다.

🔍

실전 악성코드를 잡아냅니다

암호화폐 채굴기, 리버스 셸, cron 드로퍼, SSH 키 백도어, LD_PRELOAD 루트킷, 파일리스 memfd 페이로드, 숨겨진 프로세스 — 실제로 해킹당한 서버에서 발견되는 바로 그것들입니다.

🧯

찾기만 하지 않고 고칩니다

모든 문제에는 바로 적용할 수 있는 수정이 따라옵니다: 대화형으로, --fix로 자동으로, 또는 --harden으로 새 VM 전체 셋업까지. 물론 --audit으로 100% 읽기 전용을 유지할 수도 있습니다.

🪢

당신을 서버 밖에 가두지 않습니다

방화벽을 켜기 전에 SSH 포트를 먼저 허용합니다. 키가 실제로 존재하기 전에는 키 전용 SSH 전환을 거부합니다. 새 sshd 설정은 reload 전에 검증합니다. 모든 변경은 백업되며 --rollback으로 되돌릴 수 있습니다.

🤝

대형 스캐너들과도 잘 어울립니다

ClamAV, rkhunter, chkrootkit이 설치되어 있으면 자동으로 감지해 함께 활용하고, 플래그 하나면 설치까지 해 줍니다. antivirus.sh를 빠른 1차 방어선이자 정밀 스캐너들을 묶어 주는 접착제로 사용하세요.

전체 검사 카탈로그

여섯 영역에 걸친 70가지 이상의 개별 검사. 모든 결과는 OK / INFO / WARN / CRIT로 분류되어 최종 0–100 보안 점수에 반영됩니다.

01 악성코드 & 루트킷

  • 알려진 채굴기·악성코드 프로세스 이름(xmrig, kinsing, kdevtmpfsi…)과 마이닝 풀 명령행
  • /tmp, /var/tmp, /dev/shm에서 실행 중인 프로세스
  • 바이너리가 삭제된 채 돌고 있는 프로세스, 파일리스 memfd 실행 파일
  • 숨겨진 프로세스 탐지 (readdir 은닉 방식 커널 루트킷 테스트)
  • 알려진 루트킷(Diamorphine, Reptile…)과 일치하는 커널 모듈
  • /etc/ld.so.preload & 전역 LD_PRELOAD 유저랜드 루트킷
  • tmp 디렉터리와 /dev에 숨겨진 실행 파일
  • 2 MB 이하 스크립트 안의 리버스 셸·드로퍼·채굴기 패턴
  • 패키지 체크섬 대비 핵심 바이너리 무결성 (dpkg -V / rpm -V)
  • ClamAV, rkhunter, chkrootkit 연동 (선택 사항)

02 지속성 & 백도어

  • 시스템·사용자 crontab, cron.d, hourly/daily/weekly/monthly, at 작업
  • tmp에서 실행되거나 공격 패턴과 일치하는 사용자 정의 systemd 유닛
  • rc.local, init 스크립트, udev 규칙, MOTD 스크립트
  • 모든 사용자의 셸 시작 파일 (.bashrc, .profile, .zshrc…)
  • 모든 계정의 authorized_keys 점검 — 시스템 계정에 등록된 SSH 키 적발
  • 추가 UID 0 계정, NOPASSWD sudo 규칙, 구식 .rhosts 신뢰 파일
  • 서드파티 APT 저장소 검토

03 네트워크 노출

  • 수신 대기 중인 모든 TCP/UDP 포트와 소유 프로세스, 바인드 주소
  • 노출 범위 분류: 루프백 전용 vs 외부에서 접근 가능
  • 위험 서비스 적발: Telnet, FTP, r-services, SMB, RDP, VNC, rsync, NFS, rpcbind
  • 기본 설정에 인증이 없는 저장소: Redis, MongoDB, Elasticsearch, memcached
  • :2375 포트의 Docker API (즉시 원격 root) 탐지
  • 마이닝 풀/IRC 봇넷 포트로 수립된 연결
  • 프러미스큐어스 모드 인터페이스, ARP 스푸핑 징후
  • DNS 리졸버 검토 및 /etc/hosts 하이재킹 탐지

04 SSH & 계정

  • root 로그인, 패스워드 인증, 빈 패스워드, X11 포워딩

  • shadow, sudoers, SSH 호스트 키, crontab, GRUB 설정 파일 권한
  • 엄선된 화이트리스트 + 패키지 소유 증명 기반 SUID/SGID 점검

고전적인 의미의 설치는 없습니다 — 그냥 스크립트 하나니까요. 명령 세 줄이면 전체 보고서가 나옵니다.

다운로드
curl -fsSL https://raw.githubusercontent.com/TARGET_PLEVEHOLDER/antivirus.sh/main/antivirus.sh -o antivirus.sh

모드 & 옵션

스크립트 하나, 네 가지 사용법: 점검, 대화형, 자동, 전체 하드닝. 여기에 스크립팅과 cron을 위한 영역별 실행까지.

명령동작
sudo bash antivirus.sh대화형 모드 — 모든 발견 사항을 보여 주고 수정마다 개별 승인을 받습니다. 첫 실행으로 추천합니다.
--audit보고만 수행합니다. 어떤 변경도 없음을 보장 — 프로덕션에서도 안전합니다. 종료 코드 0/1/2 = 정상/경고/심각, cron과 CI에 안성맞춤입니다.
--fix안전한 수정 사항을 모두 자동 적용합니다. 위험한 수정(방화벽, SSH 인증)은 여전히 명시적 확인을 요구합니다.
--harden--fix가 하는 모든 것에 더해 새 VM 풀 키트: 방화벽, fail2ban, unattended-upgrades, sysctl 세트, auditd, 패스워드 정책, 배너, 코어 덤프 & shm 잠금, 그리고 제대로 된 관리자 계정 생성 제안까지.
--create-usersudo 사용자 생성을 단계별로 안내: 강력한 자동 생성 패스워드, SSH 키 설치, 선택적인 키 전용 SSH 전환 — 세션을 닫기 전에 테스트하라는 안내와 함께.
--scan /var/www특정 디렉터리 악성코드 검사 (웹 루트는 악성코드가 즐겨 숨는 곳입니다).
--network / --system / --malware한 영역만 실행합니다.
--quick느린 부분(전체 디스크 검사, ClamAV)을 건너뜁니다 — 몇 초 만에 끝납니다.
--full가장 깊은 모드: 파일시스템 전체 검사, 설치된 모든 패키지 검증.
--rollback마지막 실행에서 변경된 모든 파일을 복원하고 방화벽 활성화를 되돌립니다.
--install-tools시그니처 기반 정밀 검사를 위해 ClamAV, rkhunter, chkrootkit을 설치합니다.
--report file.txt · --no-color · --yes · --exclude PATH보고서 작성과 자동화를 돕는 옵션들.
보고서를 저장하는 주간 cron 점검
0 6 * * 1 root bash /opt/antivirus.sh --audit --no-color --report /var/log/antivirus-weekly.txt

스스로를 서버 밖에 가둘 수 없도록 설계되었습니다

하드닝 도구에는 자기가 앉아 있는 나뭇가지를 잘라 버리는 나쁜 버릇이 있습니다. antivirus.sh는 원격 접속을 신성불가침으로 취급합니다:

  • 방화벽 활성화 전에 SSH를 먼저 허용합니다. SSH 포트를 세 가지 방법(sshd 설정, 활성 소켓, 현재 연결)으로 감지해 ufw enable이 실행되기 전에 허용합니다.
  • 키가 없으면 키 전용 SSH를 거부합니다. PasswordAuthentication no는 sudo 권한이 있는 사용자에게 실제 authorized_keys 항목이 있을 때만 적용됩니다. root 로그인 비활성화도 마찬가지입니다.
  • sshd 설정은 reload 전에 검증합니다. 모든 변경이 sshd -t를 통과해야 하며, 실패한 설정은 자동으로 복원됩니다. 활성 SSH 세션은 절대 끊기지 않습니다.
  • 모든 것이 백업됩니다. 수정된 각 파일은 타임스탬프가 찍힌 백업 디렉터리에 목록(manifest)과 함께 복사되며, --rollback은 스크립트가 켠 방화벽을 끄는 것까지 포함해 전부 복원합니다.
  • 삭제 대신 격리합니다. 의심스러운 파일은 격리 디렉터리로 옮겨지고 모든 권한이 제거됩니다 — 증거와 되돌릴 가능성이 모두 보존됩니다.
  • 위험한 수정은 항상 묻습니다. --fix 모드에서도 원격 접속에 영향을 줄 수 있는 것은 명시적 확인(또는 문서화된, 의도적인 --yes)을 요구합니다.

레시피: 새 VM을 3분 만에

클라우드 업체에서 갓 받은 Ubuntu VM과 root 패스워드가 손에 있습니다. 전체 절차는 이것이 전부입니다:

1단계 — 내려받고 하드닝
curl -fsSL https://raw.githubusercontent.com/TARGET_PLEVEHOLDER/antivirus.sh/main/antivirus.sh -o antivirus.sh
sudo bash antivirus.sh --harden

실행 중에 다음을 수행합니다: SSH를 미리 허용한 채 방화벽 활성화 → fail2ban & unattended-upgrades 설치 → 16개 커널 하드닝 키 적용 → SSH 강화 (검증 완료, 세션 안전) → 패스워드 정책 설정 → /dev/shm, 코어 덤프, 유휴 셸 잠금 → SSH 키를 갖춘 개인 sudo 계정 생성 제안 → 그리고 키가 자리를 잡은 뒤에야 SSH 키 전용 전환과 root 로그인 비활성화를 제안합니다. 새 로그인은 두 번째 터미널에서 테스트하세요 — 현재 세션은 그동안 내내 살아 있습니다.

FAQ

ClamAV 같은 "진짜" 안티바이러스인가요?

서로 다른 계층입니다. ClamAV는 수백만 개의 파일 시그니처를 대조하고, antivirus.sh는 실제로 리눅스 서버를 침해하는 패턴 — 채굴기, 리버스 셸, cron 드로퍼, SSH 백도어, 루트킷 흔적 — 을 사냥하며 그것들이 들어올 수 있게 만든 설정을 점검합니다. 둘은 서로를 보완하는 관계라서, antivirus.sh는 ClamAV를 감지해 있으면 활용하고 --install-tools로 설치까지 대신해 줍니다.

아주 오래된 Ubuntu 14.04나 갓 나온 최신 릴리스에서도 동작하나요?

네 — 그것이 바로 설계 제약 조건입니다. 순수 Bash에 컴파일된 부분이 없고, 모든 검사에 폴백(ss/netstat, systemd/sysvinit, ufw/iptables)이 있으며, 해당 기능이 없는 시스템에서는 메모를 남기고 자연스럽게 건너뜁니다. 같은 스크립트가 Ubuntu 12.04 시절 시스템부터 최신 릴리스까지 그대로 동작합니다.

프로덕션에서 실행해도 안전한가요?

--audit 모드는 엄격하게 읽기 전용이며, 어떤 환경에서든 권장되는 첫 실행입니다. 변경을 가하는 모드는 단계마다 확인을 받거나(대화형) 안전한 수정으로만 스스로를 제한하며(--fix), 손댄 모든 파일을 백업하고 --rollback을 제공합니다.

정말 SSH 밖으로 쫓겨날 일이 없나요?

위험한 전환들 — 방화벽 활성화, 패스워드 인증 비활성화, root 로그인 비활성화 — 은 모두 보호 장치가 있습니다: SSH 포트는 미리 허용되고, 키 전용 모드 전에 키의 존재가 검증되며, 설정은 sshd -t로 검사되어 실패 시 자동 원복되고, 현재 세션은 절대 끊기지 않습니다. 그래도 새 로그인은 두 번째 터미널에서 테스트하시길 권합니다 — 스크립트가 정확히 적절한 순간에 다시 알려 줍니다.

CRITICAL이 떴습니다. 이제 어떻게 하죠?

발견 사항을 읽어 보세요 — 각 항목이 스스로를 설명하고 증거를 출력합니다. 악성코드 발견이라면 제안되는 수정으로 격리한 뒤, 어떻게 들어왔는지(노출된 서비스? 도난당한 키? 오래된 CMS?) 추적하세요. 중요: 커널 루트킷 징후(숨겨진 프로세스, 루트킷 모듈)가 있다면 라이브 시스템에서 하는 어떤 정리 작업도 신뢰할 수 없습니다 — 깨끗한 이미지로 재구축하고 데이터만 복원하세요.

외부로 뭔가 전송하지는 않나요?

아니요. 텔레메트리도, 업로드도 없으며, 직접 요청한 것(패키지 설치, ClamAV 시그니처 업데이트) 외에는 외부 호출이 없습니다. 보고서는 디스크에만 남습니다. 동작 전체를 직접 감사할 수 있습니다 — 읽기 쉬운 Bash 파일 하나니까요.

Lynis와는 뭐가 다른가요?

Lynis는 훌륭한 순수 감사 도구로, 무엇이 잘못됐는지 알려 줍니다. antivirus.sh는 주관이 뚜렷하고 직접 손을 씁니다: 활동 중인 악성코드까지 사냥하고, 찾은 것을 안전하게, 롤백 가능하게 고칩니다. 원한다면 둘 다 돌리세요 — 두 도구의 의견이 갈리는 일은 거의 없습니다.

정기적으로 돌리려면 어떻게 하나요?

--audit --no-color --report를 붙인 cron 한 줄을 추가하세요(사용법 참고). 종료 코드 0/1/2 덕분에 CI, cron 메일, 어떤 모니터링 시스템에서든 손쉽게 알림을 걸 수 있습니다.
