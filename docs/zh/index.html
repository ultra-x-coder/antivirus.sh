为什么选择 antivirus.sh

大多数 Linux 机器都是服务器，而它们被攻破的方式总是那几个老掉牙的入口：暴露的端口、密码登录的 SSH、被遗忘的 cron 任务、没打补丁的软件包。antivirus.sh 几分钟内就能关上这些大门——并顺便排查是否已经有人溜了进来。

完整检查目录

六大领域、70 多项独立检查。每条结果都会被归类为 OK / INFO / WARN / CRIT，并计入最终的 0–100 安全评分。

01 恶意软件与 rootkit

  • 已知挖矿与恶意进程名（xmrig、kinsing、kdevtmpfsi……）及矿池命令行特征
  • 从 /tmp、/var/tmp、/dev/shm 运行的进程
  • 二进制文件已被删除的进程；无文件 memfd 可执行体
  • 隐藏进程检测（readdir 隐藏型内核 rootkit 测试）
  • 与已知 rootkit（Diamorphine、Reptile……）匹配的内核模块
  • /etc/ld.so.preload 与全局 LD_PRELOAD 用户态 rootkit
  • tmp 目录与 /dev 中的隐藏可执行文件
  • 2 MB 以内脚本中的反弹 shell / dropper / 挖矿特征
  • 核心二进制与软件包校验和的完整性比对（dpkg -V / rpm -V）
  • ClamAV、rkhunter、chkrootkit 协同调度（可选）

02 持久化与后门

  • 系统与用户 crontab、cron.d、hourly/daily/weekly/monthly、at 任务
  • 从 tmp 目录执行或匹配攻击特征的自定义 systemd 单元
  • rc.local、init 脚本、udev 规则、MOTD 脚本
  • 所有用户的 shell 启动文件（.bashrc、.profile、.zshrc……）
  • 审计每个账户的 authorized_keys——系统账户上的 SSH 密钥会被标记
  • 多余的 UID 0 账户、NOPASSWD sudo 规则、遗留的 .rhosts 信任文件
  • 第三方 APT 仓库审查

03 网络暴露面

  • 每个监听的 TCP/UDP 端口及其所属进程与绑定地址
  • 暴露面分级：仅回环 vs 公网可达
  • 标记危险服务：Telnet、FTP、r-services、SMB、RDP、VNC、rsync、NFS、rpcbind
  • 默认无鉴权的数据存储：Redis、MongoDB、Elasticsearch、memcached
  • 检测 :2375 上的 Docker API（等同即时远程 root）
  • 与矿池 / IRC 僵尸网络端口建立的连接
  • 混杂模式网卡、ARP 欺骗迹象
  • DNS 解析器审查与 /etc/hosts 劫持检测

04 SSH 与账户

  • root 登录、密码认证、空密码、X11 转发
  • MaxAuthTries、登录宽限期、keep-alive、基于主机的认证、rhosts
  • fail2ban 检测并自动配置 sshd jail
  • btmp 登录失败记录中的暴力破解痕迹
  • 空密码账户（可一键锁定）、重复 UID
  • 带登录 shell 的系统账户、sudo 组成员审查
  • 密码有效期策略与 pam_pwquality 强度要求

05 内核与文件系统

  • 16 个 sysctl 加固项：ASLR、syncookies、重定向、源路由、rp_filter、ptrace scope、dmesg/kptr 限制、protected links……
  • shadow、sudoers、SSH 主机密钥、crontab、GRUB 配置的权限
  • 基于精选白名单的 SUID/SGID 审计 + 软件包归属验证
  • 全局可写文件与未设置粘滞位的全局可写目录
  • 无主文件、immutable 标志（chattr +i）型恶意软件锁定
  • /dev/shm 与 /tmp 挂载选项（noexec、nosuid、nodev）
  • core dump 限制与 SUID dump 防护

06 平台与服务

  • 待安装的安全更新、unattended-upgrades、EOL 版本检测
  • 内核补丁后的 reboot-required 检测
  • AppArmor / SELinux 强制访问控制状态
  • auditd 取证审计、journald 持久化日志
  • NTP 时间同步（TLS 与取证分析都依赖它）
  • CPU 漏洞缓解措施（Meltdown/Spectre 家族）
  • Docker：socket 权限、特权容器、docker 组成员
  • Secure Boot 状态、GRUB 密码建议

安装与首次运行

谈不上传统意义的「安装」——它就是一个脚本。三条命令，一份完整报告到手。

下载
curl -fsSL https://raw.githubusercontent.com/TARGET_PLEVEHOLDER/antivirus.sh/main/antivirus.sh -o antivirus.sh
1 —— 安全的只读审计（不做任何改动）
sudo bash antivirus.sh --audit
2 —— 交互模式：逐项审阅并确认修复
sudo bash antivirus.sh
3 —— 或者一站式加固全新 VM
sudo bash antivirus.sh --harden

提示：没有 root 也能运行——它会执行所有可执行的检查，并明确标注其余部分。也可以 git clone：git clone https://github.com/TARGET_PLEVEHOLDER/antivirus.sh

模式与选项

一个脚本，四种用法：审计、交互、自动、全面加固。另有面向脚本化与 cron 的细分范围。

命令行为
sudo bash antivirus.sh交互模式——每条发现都会展示，每项修复都需单独确认。建议首次运行使用。
--audit仅生成报告。保证零改动——生产环境也可放心运行。退出码 0/1/2 = 干净/警告/严重，非常适合 cron 与 CI。
--fix自动应用所有安全的修复。高风险修复（防火墙、SSH 认证）仍需明确确认。
--harden包含 --fix 的全部内容，外加整套新 VM 装备：防火墙、fail2ban、unattended-upgrades、sysctl 套件、auditd、密码策略、登录横幅、core dump 与 shm 锁定，并可顺带创建一个正经的管理员用户。
--create-user引导式创建 sudo 用户：自动生成强密码、安装 SSH 密钥、可选切换到仅密钥 SSH——并提醒你在关闭当前会话前先测试。
--scan /var/www对指定目录做恶意软件扫描（Web 根目录是最受欢迎的藏身处）。
--network / --system / --malware只运行某一类检查。
--quick跳过耗时部分（全盘扫描、ClamAV）——几秒钟完成。
--full最深入的模式：扫描整个文件系统，校验每一个已安装软件包。
--rollback恢复上次运行修改的所有文件，并撤销防火墙启用。
--install-tools安装 ClamAV、rkhunter 和 chkrootkit，获得基于特征库的扫描深度。
--report file.txt · --no-color · --yes · --exclude PATH报告与自动化辅助选项。
每周 cron 审计并保存报告
0 6 * * 1 root bash /opt/antivirus.sh --audit --no-color --report /var/log/antivirus-weekly.txt

从设计上就不让你把自己锁在门外

加固工具有个坏毛病：专锯你正坐着的那根树枝。antivirus.sh 把远程访问当作不可触碰的底线：

  • 启用防火墙前先放行 SSH。通过三种途径探测你的 SSH 端口（sshd 配置、实时套接字、当前连接），并在 ufw enable 执行之前放行。
  • 没有密钥就拒绝仅密钥 SSH。只有当具备 sudo 权限的用户确实拥有 authorized_keys 条目时，才会应用 PasswordAuthentication no。禁用 root 登录同理。
  • sshd 配置在重载前必经校验。每次改动都要通过 sshd -t；校验失败的配置会自动还原。已建立的 SSH 会话绝不掉线。
  • 所有改动都有备份。每个被修改的文件都会复制到带时间戳的备份目录并记录清单；--rollback 可全部还原，包括关闭由脚本启用的防火墙。
  • 只隔离，绝不删除。可疑文件会被移入隔离目录并剥夺全部权限——既保全证据，又可随时恢复。
  • 高风险修复永远先问。即使在 --fix 模式下，任何可能影响远程访问的操作都需要明确确认（或使用文档中明示的、需主动加上的 --yes）。

实战配方：3 分钟搞定全新 VM

你刚从云服务商那里拿到一台全新的 Ubuntu 虚拟机，手里攥着 root 密码。完整流程就这一套：

第 1 步 —— 下载并加固
curl -fsSL https://raw.githubusercontent.com/TARGET_PLEVEHOLDER/antivirus.sh/main/antivirus.sh -o antivirus.sh
sudo bash antivirus.sh --harden

运行过程中它会：启用防火墙并预先放行 SSH → 安装 fail2ban 与 unattended-upgrades → 应用 16 个内核加固项 → 收紧 SSH（经过校验、会话安全）→ 设置密码策略 → 锁定 /dev/shm、core dump 与闲置 shell → 提议创建带 SSH 密钥的个人 sudo 用户 → 直到密钥就位之后，才会提议把 SSH 切换为仅密钥模式并禁用 root 登录。请在另一个终端测试新登录——你当前的会话全程保持在线。

FAQ

它是像 ClamAV 那样的「真」杀毒软件吗？

两者属于不同的层面。ClamAV 匹配数百万条文件特征；antivirus.sh 追踪的是真正攻陷 Linux 服务器的那些手法——挖矿程序、反弹 shell、cron dropper、SSH 后门、rootkit 痕迹——并审计放它们进门的配置。它们互为补充，所以 antivirus.sh 会检测 ClamAV、在它存在时加以调用，还能通过 --install-tools 帮你装上。

我那台老古董 Ubuntu 14.04 / 最新发行版能跑吗？

能——这本身就是一条设计约束。纯 Bash、没有编译产物，每个探测都有后备方案（ss/netstat、systemd/sysvinit、ufw/iptables），遇到不存在的功能会优雅跳过并附上说明。同一个脚本可以从 Ubuntu 12.04 时代的系统一路跑到最新版本。

在生产环境运行安全吗？

--audit 模式严格只读，是任何环境下都推荐的首次运行方式。会做改动的模式要么逐步确认（交互式），要么只限于安全修复（--fix），所有被改动的文件都有备份，并提供 --rollback。

它真的不会把我锁在 SSH 外面？

那些危险的切换——启用防火墙、禁用密码认证、禁用 root 登录——全都有防护：SSH 端口预先放行、切换仅密钥模式前先验证密钥存在、配置经 sshd -t 校验且失败自动回退、当前会话绝不会被断开。当然仍建议你在第二个终端测试新登录；脚本会在恰到好处的时机提醒你。

它报了 CRITICAL，现在怎么办？

仔细读那条发现——每条都自带解释并打印证据。对于恶意软件类发现，先用提供的修复完成隔离，再追查它是怎么进来的（暴露的服务？被盗的密钥？过时的 CMS？）。重要提示：如果迹象指向内核 rootkit（隐藏进程、rootkit 模块），在带毒系统上做的任何清理都不可信——请用干净镜像重建，只恢复数据。

它会回传数据或上传任何东西吗？

不会。没有遥测、没有上传，除了你主动要求的操作（安装软件包、更新 ClamAV 特征库）之外没有任何外部请求。报告只保存在你的磁盘上。它的全部行为都可审计——毕竟就是一个可读的 Bash 文件。

它和 Lynis 有什么区别？

Lynis 是一款出色的纯审计工具——它告诉你哪里有问题。antivirus.sh 更有主见、也更动手：它还会追猎活跃的恶意软件，并安全地修复所发现的问题，且支持回滚。愿意的话两个都跑；它们的结论几乎不会互相矛盾。

如何定期运行？

加一条 cron（见「用法」），带上 --audit --no-color --report。退出码 0/1/2 让你能轻松地从 CI、cron 邮件或任意监控系统发出告警。
