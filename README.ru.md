`antivirus.sh` — это самодостаточный Bash-скрипт для Linux-серверов. Он проверяет вредоносные файлы, точки закрепления, процессы, целостность пакетов, базовые сетевые индикаторы и безопасно отправляет подозрительные файлы в карантин. В этом репозитории находится только антивирусная часть, выделенная из более крупного пакета `antivirus.sh`.

Структура README сделана с оглядкой на стиль оформления эталонного проекта: <https://github.com/ultra-x-coder/antivirus.sh/>

## Быстрый старт

```bash
chmod +x antivirus.sh

# интерактивная проверка: показать находки и спросить перед исправлением
sudo bash antivirus.sh

# только аудит: ничего не менять
sudo bash antivirus.sh --audit

# автоматически применить безопасные исправления
sudo bash antivirus.sh --fix

# проверить конкретный путь
sudo bash antivirus.sh --scan /var/www

# самый глубокий режим проверки
sudo bash antivirus.sh --full
```

Лучше запускать от `root`. От обычного пользователя скрипт тоже работает, но с урезанным покрытием и пользовательскими путями для отчета и карантина.

## Что проверяет скрипт

### Вредоносные файлы и подозрительный контент

- Реверс-шеллы, дропперы, майнеры и типовые бэкдор-шаблоны в скриптах и текстовых файлах
- Скрытые или исполняемые полезные нагрузки в `/tmp`, `/var/tmp` и `/dev/shm`
- Подозрительные файлы в путях, переданных через `--scan`
- Опциональные глубокие проверки через ClamAV, если он установлен

### Процессы и признаки активности

- Известные имена вредоносных процессов и майнеров
- Подозрительные командные строки вроде `xmrig`, `minerd`, `kdevtmpfsi`, `kinsing` и похожих семейств
- Бинарники, запущенные из временных директорий
- Безфайловые `memfd`-процессы
- Индикаторы скрытых процессов и руткитов
- Подозрительные модули ядра, связанные с известными Linux-руткитами

### Закрепление и бэкдоры

- `/etc/ld.so.preload`
- Cron-задачи и планировщики
- Юниты systemd
- `rc.local`
- Файлы инициализации оболочки
- Правила udev
- Подозрительные записи в `authorized_keys`
- Сторонние репозитории пакетов

### Сеть и целостность

- Исходящие соединения на порты, часто используемые майнинг-пулами или IRC-ботнетами
- Проверка целостности критичных системных бинарников через `dpkg -V` или `rpm -V`
- Опциональный запуск `rkhunter` и `chkrootkit`, если они установлены

## Режимы

| Команда | Что делает |
| --- | --- |
| `sudo bash antivirus.sh` | Интерактивная проверка. Каждое исправление подтверждается. |
| `sudo bash antivirus.sh --audit` | Только отчет. Изменения не вносятся. |
| `sudo bash antivirus.sh --fix` | Автоматически применяет безопасные исправления. Опасные действия могут потребовать подтверждения. |
| `sudo bash antivirus.sh --install-tools` | Устанавливает ClamAV, rkhunter и chkrootkit через пакетный менеджер. |

## Опции

| Опция | Описание |
| --- | --- |
| `--scan PATH` | Проверить конкретную директорию. Можно указывать несколько раз. |
| `--exclude PATH` | Исключить путь из проверки. Можно указывать несколько раз. |
| `--quick` | Пропустить медленные проверки: ClamAV, rkhunter и тест скрытых процессов. |
| `--full` | Выполнить максимально глубокое сканирование, включая более широкий охват файловой системы. |
| `--yes`, `-y` | Отвечать yes на все вопросы, включая рискованные действия. |
| `--no-external` | Не запускать и не предлагать сторонние сканеры. |
| `--report FILE` | Записать отчет в указанный файл. |
| `--no-color` | Отключить цветной вывод. |
| `--version` | Показать текущую версию. |
| `--help` | Показать встроенную справку. |

## Модель безопасности

- Подозрительные файлы помещаются в карантин, а не удаляются.
- Режим аудита безопасен для проверки production-хостов.
- По умолчанию используется интерактивный режим.
- Автоисправления ограничены тем, что скрипт считает безопасным.
- Рискованные действия явно помечаются, если не использовать `--yes`.

## Вывод и артефакты

- Коды выхода: `0` чисто, `1` предупреждения, `2` критические находки
- Отчеты для `root`: `/var/log/antivirus-whole/`
- Карантин для `root`: `/var/lib/antivirus-whole/quarantine/`
- Отчеты без `root`: `~/.antivirus-whole/log/`
- Карантин без `root`: `~/.antivirus-whole/quarantine/`

Действия с карантином пишутся в `quarantine.log`. Восстановление файла выполняется вручную после проверки.

## Требования

- Bash в Linux
- Для полного покрытия желательно запускать от `root`
- Обязательных внешних зависимостей нет
- Опциональные инструменты: ClamAV, rkhunter, chkrootkit

## Ограничения

- Скрипт рассчитан прежде всего на Linux-серверы
- Вне Ubuntu/Debian-подобных систем работает в режиме best effort
- Это эвристический сканер, а не замена hardening, EDR или постоянному мониторингу
- Проверки без сигнатур могут давать ложные срабатывания, их нужно разбирать вручную
